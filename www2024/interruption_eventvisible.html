📝 논문

✍️ 문제 제기

  • 클라우드 제공업체들은 수요가 낮을 때 남는 컴퓨팅 자원을 스팟 인스턴스라는 형태로 할인된 가격에 제공함
  • 이는 온디맨드 가격보다 최대 90% 저렴할 수 있음
  • 그러나 스팟 인스턴스는 수요 변화에 따라 갑자기 중단될 수 있어 기존 애플리케이션에 사용이 어려움
  • 클라우드 제공업체들은 사용자들이 스팟 인스턴스를 더 효과적으로 사용할 수 있도록 다양한 데이터셋을 제공
  • 하지만 스팟 인스턴스 데이터셋의 효과는 정량적으로 평가되지 않았으며, 많은 사용자들이 여전히 추측에 의존해 스팟 인스턴스를 선택하고 있음

🚧 핵심 기여

  • AWS, Azure, GCP의 스팟 인스턴스 신뢰성 비교 분석
  • 스팟 인스턴스 데이터셋의 철저한 분석
  • 스팟 인스턴스 데이터셋 값 예측의 효과성 정량적 제시
  • 스팟 인스턴스의 중단 확률을 낮출 수 있는 방법론 제안
    • 제안된 방법을 이용하면 기존 방법보다 스팟 인스턴스의 생존시간을 최대 168%개선

📣 발표영상

👨🏻‍💻담당 업무

  • 스팟 데이터 분석 및 시각화
    • SpotLake에서 2022년 11월부터 2023년 8월까지의 데이터 이용
    • AWS, Azure, GCP의 가격, IF, SPS 데이터셋 분석
    • 각 데이터의 분포, 변화패턴 등 시각화
    • AWS SPS데이터가 일일 변화 패턴을 보임을 발견
  • sklearn과 XGboost를 이용한 스팟 가용성 데이터 예측
    • AWS SPS데이터를 예측하기 위해 다양한 머신러닝 모델(Linear Regressor, Random Forest, XGBoost 등) 적용
    • F-1 점수를 이용한 모델 예측 성능 평가
    • XGboost가 가장 좋은 성능을 보임(0.97)
  • 스팟 인스턴스 중단 실험 진행
    • 여러 클라우드 제공업체의 스팟 인스턴스 중단 데이터를 수집할 수 있는 코드 작성
    • 실험 방법
      • 선택된 각 스팟 인스턴스를 실행
      • 중단 이벤트가 발생할 때까지 실행 지속
      • 중단 발생 시 이벤트를 기록
      • 중단 후 다시 인스턴스가 사용 가능할 때까지 스팟 인스턴스 요청을 계속 시도
  • 중단 데이터 분석
    • Kaplan-Meier Estimator를 사용하여 생존율 분석
      • 여러 벤더와 데이터셋 값에 따른 생존율 비교
        • AWS가 가장 낮은 생존율을 보임 (중앙값 1.2시간)
        • GCP와 Azure는 24시간 실험 동안 절반 이상이 중단되지 않음
      • AWS의 경우:
        • 높은 SPS 점수를 가진 인스턴스가 더 긴 생존 시간을 보임
        • 평균적으로 SPS 3점 인스턴스는 4.7시간, 1점 인스턴스는 1.8시간 실행됨
    • 누적 분포 함수를 이용한 예측 SPS에 따른 생존시간 비교
      • 초기 높은 SPS이더라도 미래 예측값을 고려하면 63.2% 더 긴 생존 시간 확보 가능
      • 초기 낮은 SPS에도 불구하고 예측값이 높은 경우 168% 더 긴 생존 시간 확보 가능
